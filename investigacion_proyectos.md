GitHub Projects y la gestion de proyectos
GitHub Projects es una herramienta integrada en GitHub para planificar y
hacer seguimiento del trabajo de desarrollo de software.

Diferencias con un tablero Kanban tradicional:
  - Integracion nativa con issues, PRs y commits del repositorio
  - Automatizacion: issues se mueven automaticamente al cerrarlos
  - Multiples vistas: tablero, tabla, hoja de ruta (roadmap)
  - Filtros y agrupaciones por etiquetas, asignados, milestones

Automatizaciones posibles en GitHub Projects:
  1. Al crear un issue -> se agrega automaticamente a 'Todo'
  2. Al cerrar un issue -> se mueve automaticamente a 'Done'
  3. Al fusionar un PR -> se actualiza el estado del issue relacionado
